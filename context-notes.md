# 관리자 대시보드 작업 맥락 노트

## 2026-05-18

- 관리자 기능은 한 번에 한 페이지씩 진행한다.
- 1차 완료 기준은 백엔드 API, 관리자 로그인, `/admin` 최소 화면 연결까지다.
- 관리자 코드는 MVP 시연용으로 `admin2026` 고정값을 사용한다.
- 대시보드 데이터는 실제 DB만 사용하고, 샘플 fallback은 두지 않는다.
- 대시보드 레이아웃은 답변 대기 문의를 크게 배치하는 문의 우선형으로 결정했다.
- `/admin/agendas`, `/admin/inquiries`는 이번 단계에서 페이지를 만들지 않고 링크만 남긴다.
- 기존 미추적 `AGENTS.md`는 사용자 작업으로 보고 건드리지 않는다.
- 백엔드 헬퍼 테스트는 먼저 작성했고, `backend/lib/adminDashboard` 모듈 없음으로 실패하는 RED 상태를 확인했다.
- `buildDashboardData`는 DB query 인터페이스를 받아 KPI, 목록, 차트 데이터를 한 번에 정규화한다.
- `/api/admin/dashboard`는 DB 연결 실패 시 한국어 일반 오류 메시지를 반환한다.
- 브라우저에서 `/admin/login`에 접속해 `admin2026` 입력 후 `/admin`으로 이동하는 흐름을 확인했다.
- 로컬 DB 인증 정보가 없어 실제 데이터 렌더링 대신 API 오류 상태가 표시되는 것까지 확인했다.
- 사용자가 보낸 참고 이미지는 대시보드가 아니라 관리자 안건 관리 화면이다.
- `/admin/agendas`를 추가해 검은 관리자 GNB, 크림색 배경, 상태 필터 칩, 검색 박스, 둥근 테이블, 상태 변경 드롭다운을 참고 이미지 톤으로 맞췄다.
- 관리자 로그인 성공 후 첫 이동 경로를 `/admin/agendas`로 바꿔 사용자가 참고 이미지와 같은 화면을 먼저 보도록 했다.
- 두 번째 참고 이미지는 관리자 대시보드 화면이다.
- `/admin` 대시보드를 KPI 4개, 공감도 높은 안건, 답변 필요한 문의, 하단 3개 통계 카드 구조로 재배치했다.
- DB 연결 실패 시에도 샘플 데이터는 쓰지 않고 0값과 빈 상태로 대시보드 레이아웃을 볼 수 있게 했다.
- 대시보드 디자인 반영 후 로그인 성공 경로는 다시 `/admin`으로 복구했다.
- 다음 관리자 기능은 명세서 4.3의 `/admin/inquiries` 문의 관리로 진행한다.
- 답변 없이 상태만 변경하는 별도 API는 이번 MVP에서 만들지 않고, 답변 저장 시 선택한 상태로 함께 반영한다.
- 문의 관리 화면은 기존 관리자 GNB와 대시보드/안건 관리의 크림색 배경, 흰 패널, 파란 선택 강조 톤을 유지한다.
- `/api/admin/inquiries`는 전체/상태 필터 목록과 상태별 카운트를 반환하고, 답변 대기 → 검토 중 → 답변 완료 순으로 정렬한다.
- `/api/admin/inquiries/:id/reply`는 빈 답변을 거부하고, 답변 저장 시 `status`와 `replied_at`을 함께 갱신한다.
