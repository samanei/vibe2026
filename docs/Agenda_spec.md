# 공개 안건 기능명세서

## 2.1 안건 목록

### 2.1.1 상단 GNB
- 로고(캠퍼스보이스) / '내 문의함' 버튼 / '비공개 문의' 버튼 / '+ 안건 등록' 버튼
- 모든 학생 화면 공통 상단 고정 (sticky)
- **수용 기준**
  - 모든 페이지 최상단 sticky 표시
  - 각 버튼 클릭 시 해당 화면으로 이동
- **사용자**: 전체 사용자
- **중요도**: 높음

### 2.1.2 좌측 사이드바 — 카테고리 필터
- 카테고리 목록: 전체 / 시설 / 수업 / 행정 / 복지 / 학식 / 행사 / 학생회 / 기타
- 선택 시 해당 카테고리 안건만 표시
- **수용 기준**
  - 클릭 시 목록 즉시 필터링
  - 선택된 항목 강조 스타일
  - '전체' 선택 시 모든 안건 표시
- **엣지 케이스**: 새 안건 등록 시 카운트 즉시 업데이트
- **비고**: 카테고리별 안건 수 실시간 표시
- **중요도**: 높음

### 2.1.3 좌측 사이드바 — 이번 주 현황
- 신규 안건 / 처리 완료 / 검토 중 / 총 투표 수 통계를 카드 형태로 표시
- 주 기준: 월요일 기준 리셋
- **수용 기준**: 주 단위 집계로 표시
- **엣지 케이스**: 주 초기화 시점 처리(월요일 기준), 실시간 업데이트 vs 주기적 캐시 정책 결정
- **중요도**: 중간

### 2.1.4 탭 (공개 안건 / 비공개 문의 작성)
- 공개 안건 탭(기본 선택) / 비공개 문의 작성 탭
- 비공개 문의 탭 클릭 시 문의 작성 화면으로 이동
- **수용 기준**
  - 기본값: '공개 안건' 탭 활성
  - 선택된 탭 하단 진한 테두리 강조
- **중요도**: 높음

### 2.1.5 인라인 카테고리 칩 필터
- 탭 하단 가로 스크롤 칩: 전체 / 시설 / 수업 / 행정 / 복지 / 학식 / 행사 / 학생회
- 좌측 사이드바 카테고리 필터와 연동
- **수용 기준**
  - 선택 시 해당 카테고리만 노출
  - 사이드바 카테고리와 동기화
  - '전체' 칩 기본 선택
- **엣지 케이스**: 칩과 사이드바 동시 조작 시 마지막 조작 기준으로 반영
- **중요도**: 높음

### 2.1.6 정렬 버튼
- 🔥 인기순(기본, 찬성 수 기준) / 🕐 최신순 / 👍 찬성률순 버튼
- **수용 기준**
  - 기본값: 인기순(찬성 수 기준)
  - 선택 시 목록 즉시 재정렬
  - 버튼 3개 중 항상 1개 활성 상태 유지
- **엣지 케이스**: 동일 찬성 수일 때 보조 정렬 기준 정책 결정
- **중요도**: 높음

### 2.1.7 안건 카드
- 카드 구성:
  - 카테고리 배지
  - 처리 상태 배지 (색상 구분)
  - HOT 배지 (조건부 표시)
  - 익명 · 학과명 (우측)
  - 안건 제목 (굵게)
  - 본문 미리보기 (2줄 말줄임)
  - 찬성 수 / 반대 수 / 공감률 바 / 공감률%
  - 등록일
- **수용 기준**
  - 카드 클릭 시 안건 상세 화면으로 이동
  - 공감률 = 찬성/(찬성+반대)*100 계산
  - 등록일 YYYY.MM.DD 형식
- **엣지 케이스**: 본문 없는 안건은 제목만 표시, 삭제된 안건 링크 진입 시 '삭제된 안건입니다' 안내
- **중요도**: 높음

### 2.1.8 처리 상태 배지 색상
| 상태 | 색상 |
|------|------|
| 접수됨 | 파랑 |
| 검토 중 | 주황 |
| 학교 전달 완료 | 보라 |
| 반영 완료 | 초록 |
| 반영 어려움 | 회색 |

- **수용 기준**: 상태별 배경색 + 텍스트색 일관 적용, 관리자 상태 변경 시 즉시 반영
- **중요도**: 높음

### 2.1.9 HOT 배지
- 일정 기준 이상 찬성 수·공감률 안건에 '🔥 HOT' 배지 자동 표시
- **수용 기준**: 기준값 TBD (찬성 수 또는 공감률 기준), 기준 충족 시 자동 표시
- **중요도**: 중간

### 2.1.10 페이지네이션
- 페이지 이동 버튼, 현재 페이지 강조
- **수용 기준**: 현재 페이지 강조 스타일, 페이지당 노출 건수 TBD
- **엣지 케이스**: 마지막/첫 페이지에서 이동 버튼 동작 없음
- **중요도**: 중간

## 2.2 안건 상세

### 2.2.1 네비게이션 및 작성자 표시
- 상단 좌측 '← 목록으로' 버튼, 우측 '익명 · 학과명' 표시
- **수용 기준**: 클릭 시 목록 화면 복귀, 목록 스크롤 위치 유지 권장
- **중요도**: 높음

### 2.2.2 안건 본문
- 카테고리 배지 + 처리상태 배지 + 등록일 표시
- 안건 제목(대형), 문제 상황 섹션(레이블+본문 전체), 개선 요청 섹션(레이블+본문 전체)
- **수용 기준**
  - 제목·문제 상황·개선 요청 전문 표시(줄임 없음)
  - 등록일 YYYY.MM.DD 형식
- **중요도**: 높음

### 2.2.3 투표 현황 — 수치 및 바
- 찬성 수 / 공감률(큰 폰트·파란색) / 반대 수 / 총 참여 수 표시
- 찬성 비율 가로 바 시각화
- **수용 기준**
  - 공감률 = 찬성/(찬성+반대)*100
  - 투표 발생 시 즉시 업데이트
  - 바 너비는 공감률에 비례
- **엣지 케이스**: 투표 수 0인 경우 공감률 표시 방법 결정 (0% 또는 '-')
- **중요도**: 높음

### 2.2.4 찬성하기 버튼
- 파란 배경 '👍 찬성하기' 버튼
- 클릭 시 찬성 투표, 1인 1회 제한, 이미 투표 시 상태 표시
- **수용 기준**
  - 로그인 사용자만 투표 가능, 미로그인 클릭 시 로그인 유도
  - 이미 찬성: 버튼 강조 상태 유지
  - 이미 반대 투표 상태에서 찬성 클릭: 반대 취소 + 찬성으로 전환
  - 투표 즉시 수치 업데이트
- **엣지 케이스**: 네트워크 오류 시 실패 토스트 + 이전 상태 유지, 중복 투표 서버 차단
- **비고**: 중복 방지 서버 검증 필수
- **중요도**: 높음

### 2.2.5 반대하기 버튼
- 분홍 배경 '👎 반대하기' 버튼
- 찬성과 동시 선택 불가, 전환 시 기존 투표 자동 취소
- **수용 기준**: 찬성+반대 동시 선택 불가, 이미 반대 시 버튼 강조 상태 유지
- **엣지 케이스**: 네트워크 오류 시 실패 토스트 + 이전 상태 유지
- **중요도**: 높음

### 2.2.6 처리 현황 타임라인
- 세로 타임라인으로 처리 단계 시각화:
  - 완료 단계: 채워진 점 + 날짜
  - 현재 단계: 강조 점 + 날짜 + 학생회 코멘트
  - 미완료 단계: 빈 점
- 단계: 접수됨 → 검토 중 → 학교 전달 완료 → 반영 완료 / 반영 어려움
- **수용 기준**: 관리자 상태 변경 시 즉시 반영, 코멘트 없는 단계는 날짜만 표시
- **중요도**: 높음

### 2.2.7 안건 정보 사이드바
- 우측 카드: 카테고리 / 등록 학과 / 등록일 / 총 투표 수 / 공감률 (파란색 강조)
- **수용 기준**: 읽기 전용 메타 정보, 투표 수 실시간 반영
- **중요도**: 중간

## 2.3 안건 작성

### 2.3.1 취소 버튼 및 등록하기 버튼
- 상단 좌측 '← 취소', 우측 '등록하기' 버튼
- 등록하기는 필수 항목 모두 입력 시 활성화
- **수용 기준**
  - 취소 클릭: 입력 중 내용 있으면 확인 모달 노출 후 이전 화면 이동
  - 등록하기: 카테고리·제목·문제상황·개선요청 모두 입력 시 활성
- **엣지 케이스**: 이미 제출 중 중복 클릭은 첫 요청만 처리
- **중요도**: 높음

### 2.3.2 익명 등록 안내 배너
- 파란 배경 배너: '🔒 익명으로 등록됩니다 — 이름·학번은 절대 공개되지 않아요. 학과 정보만 표시됩니다.'
- **수용 기준**: 정적 안내 배너, 항상 표시
- **중요도**: 높음

### 2.3.3 카테고리 선택 칩
- 시설 / 수업 / 행정 / 복지 / 학식 / 행사 / 학생회 / 기타 칩 버튼
- 1개 선택 필수(*), 선택된 칩 파란 배경 강조
- **수용 기준**: 필수 항목, 미선택 시 등록 불가, 단일 선택
- **중요도**: 높음

### 2.3.4 안건 제목 입력 필드
- 텍스트 입력 필드, 최대 50자, 필수(*)
- **수용 기준**: 최대 50자 초과 차단 또는 오류 표시, 미입력 시 등록 불가
- **엣지 케이스**: 공백만 입력 시 제출 시 유효성 검증으로 차단
- **비고**: 글자 수 카운터 권장
- **중요도**: 높음

### 2.3.5 문제 상황 입력
- 현재 불편 상황 서술, 필수(*), 최대 글자 수 TBD
- **수용 기준**: 필수 항목, 미입력 시 등록 불가
- **중요도**: 높음

### 2.3.6 개선 요청 입력
- 원하는 개선 방향 서술, 필수(*), 최대 글자 수 TBD
- **수용 기준**: 필수 항목, 미입력 시 등록 불가
- **중요도**: 높음

### 2.3.7 경고 문구 및 등록 버튼
- 경고: '⚠ 욕설·비방·허위사실이 포함된 안건은 관리자가 비공개 처리할 수 있습니다.'
- 하단 '안건 등록하기' 버튼
- **수용 기준**
  - 경고 문구 항상 표시
  - 필수 항목 모두 입력 시에만 버튼 활성
  - 등록 성공: 목록 이동 + 성공 토스트
  - 등록 실패: 오류 메시지 표시
- **엣지 케이스**: 중복 제출 방지(버튼 비활성화 후 처리), 네트워크 오류 시 오류 토스트 + 재시도
- **중요도**: 높음
