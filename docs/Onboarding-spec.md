# 온보딩 / 회원가입 기능명세서

## 1.1 이메일 인증

### 1.1.1 학교 이메일 입력
- 학교 도메인 이메일 주소 입력 화면
- 상단 스텝바(1/3) 표시
- 필드 하단 도메인 제한 안내 문구 표시
- **수용 기준**
  - 이메일 미입력 또는 허용 도메인 불일치 시 '인증 메일 받기' 버튼 비활성
  - 올바른 형식 입력 시 버튼 활성화
  - 버튼 클릭 시 인증 메일 발송 요청
- **엣지 케이스**
  - 허용 외 도메인 입력: 인라인 오류 메시지 노출
  - 빈칸 제출: 버튼 비활성화로 방지
  - 중복 발송 방지: 일정 시간 쿨타임 적용 권장
- **비고**: 도메인 화이트리스트 서버 검증 필수
- **사용자**: 비로그인 사용자
- **중요도**: 높음

### 1.1.2 인증 코드 확인
- 발송된 인증 메일의 코드 입력으로 재학생 여부 확인
- 인증 완료 시 다음 단계(학과·학년 선택)로 자동 이동
- **수용 기준**
  - 올바른 코드 입력 시 인증 완료 → 다음 단계 이동
  - 잘못된 코드: 인라인 오류 메시지
  - 코드 만료 시: 재발송 안내 및 버튼 제공
- **엣지 케이스**
  - 만료 코드 입력: 재발송 안내
  - 반복 오류: 잠금 처리 정책 결정 필요
  - 네트워크 오류: 오류 토스트 + 재시도
- **비고**: 코드 만료 시간 정책 결정 필요
- **사용자**: 비로그인 사용자
- **중요도**: 높음

### 1.1.3 개인정보 안내 배너
- 이메일 입력 화면 하단 체크 아이콘 포함 3줄 안내:
  - ✓ 이름·학번은 저장되지 않습니다
  - ✓ 학과 정보만 최소한으로 표시됩니다
  - ✓ 실제 재학생만 가입 가능합니다
- **수용 기준**: 정적 안내 문구, 이메일 인증 화면에 항상 표시
- **사용자**: 비로그인 사용자
- **중요도**: 높음

### 1.1.4 기존 계정 로그인 링크
- 이메일 인증 화면 하단 '이미 계정이 있으신가요? 로그인' 텍스트 링크
- **수용 기준**: 클릭 시 로그인 화면으로 이동
- **사용자**: 비로그인 사용자
- **중요도**: 중간

## 1.2 학과·학년 선택

### 1.2.1 상단 스텝바 (2/3)
- 2단계 진입 시 스텝 1 완료(체크) → 스텝 2 활성 상태 업데이트
- 레이블: '✓ 이메일 인증 | 2. 학과·학년 선택 | 3. 가입 완료'
- **수용 기준**
  - 이메일 인증 완료 후에만 2단계 진입 가능
  - 완료된 스텝 체크 표시, 현재 스텝 강조
- **엣지 케이스**: 인증 미완료 상태에서 URL 직접 접근 시 1단계로 리다이렉트
- **사용자**: 비로그인 사용자
- **중요도**: 높음

### 1.2.2 익명 표시 안내 배너
- 자물쇠 아이콘 포함 파란 배너: '🔒 익명으로 표시됩니다 — 예: 익명 · 컴퓨터공학과'
- **수용 기준**: 정적 안내 배너, 항상 표시. 이름·학번 비노출임을 사전 고지
- **사용자**: 비로그인 사용자
- **중요도**: 높음

### 1.2.3 학과 선택 드롭다운
- 소속 학과를 드롭다운에서 선택, 필수 항목(*)
- 학과 목록은 DB에서 동적으로 불러옴
- **수용 기준**
  - 미선택 시 '다음으로' 버튼 비활성
  - 선택 완료 시 버튼 활성화
- **엣지 케이스**
  - 학과 목록 로딩 실패: 오류 메시지 + 재시도
  - 빈 목록: '학과 정보를 불러올 수 없습니다' 안내
- **비고**: 학과 목록 DB 관리 필요
- **사용자**: 비로그인 사용자
- **중요도**: 높음

### 1.2.4 학년 선택 버튼 그룹
- 1학년 / 2학년 / 3학년 / 4학년 버튼 중 1개 선택, 필수 항목(*)
- 선택된 버튼 강조 스타일 적용
- **수용 기준**
  - 미선택 시 '다음으로' 버튼 비활성
  - 단일 선택, 재선택 시 이전 선택 해제
- **엣지 케이스**: 이미 선택된 학년 재클릭 시 선택 유지 또는 해제 정책 결정
- **사용자**: 비로그인 사용자
- **중요도**: 높음

### 1.2.5 다음으로 버튼
- 학과 + 학년 모두 선택 완료 시 활성화
- 클릭 시 서버 저장 후 온보딩 완료 화면으로 이동
- **수용 기준**
  - 두 항목 모두 선택 시에만 활성
  - 저장 실패 시 오류 토스트 + 재시도
- **엣지 케이스**: 네트워크 오류 시 오류 토스트 + 입력값 유지, 중복 제출 방지
- **사용자**: 비로그인 사용자
- **중요도**: 높음

### 1.2.6 이전으로 버튼
- 클릭 시 이메일 인증 화면으로 복귀
- **수용 기준**: 이미 완료된 이메일 인증 상태 유지
- **엣지 케이스**: 입력값(학과·학년) 초기화 또는 유지 정책 결정
- **사용자**: 비로그인 사용자
- **중요도**: 중간

## 1.3 온보딩 완료

### 1.3.1 완료 화면
- 가입 완료 안내 화면
- 체크 아이콘(초록) + '온보딩 완료!' 제목 + 환영 문구
- 개인정보 보호 안내 박스(초록)
- '시작하기' 버튼
- **수용 기준**
  - 가입 완료 후 자동 진입
  - '시작하기' 클릭 시 공개 안건 목록 메인으로 이동
  - 스텝바 3/3 완료 표시
- **엣지 케이스**: 뒤로가기 시 이미 가입 완료 상태이므로 메인으로 리다이렉트
- **사용자**: 신규 사용자
- **중요도**: 높음

## 1.4 로그인

### 1.4.1 기존 계정 로그인
- 이메일 + 비밀번호 입력 후 로그인
- 로그인 성공 시 세션 생성 후 메인으로 이동
- 기본 프로필 미완성 시 미완료 단계로 이동
- **수용 기준**
  - 로그인 성공: 세션 생성 → 메인 이동
  - 기본 프로필 미완성: 미완료 단계부터 재진입
  - 비로그인 상태에서 제한 행동 시도 후 로그인 완료 시 원래 화면 복귀
- **엣지 케이스**
  - 잘못된 비밀번호: 오류 메시지 노출
  - 탈퇴 계정 로그인 시도: 안내 메시지
  - 네트워크 오류: 오류 토스트 + 재시도
- **비고**: JWT 또는 세션 쿠키
- **사용자**: 비로그인 사용자
- **중요도**: 높음
